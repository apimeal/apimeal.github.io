L'explication

L'APImeal se divise en 5 grandes parties représentant les principales étapes réalisé par le serveur HTTP.
Chaque partie est composée d'un module correspondant à l'action principale et de un ou deux point(s) d'encrage(s) (hook).
Chaque hook sert à effectuer de potentiels actions de pré ou post traitement.

Connection

Ce module permet de gérer la connection d'un client et de récupérer les requètes envoyées

Nom Paramètre(s) Description Exemple
preConnection IConnexion *, Error & Appelé quand un client se connecte Blacklist
connection Accepte la connexion d’un client
postConnection IConnexion *, Error & Gestion SSL SSL

ParseRequest

Ce module permet de parser les requètes envoyées ainsi que de les décoder et d'effectuer les potentiels URL Rewriting

Nom Paramètre(s) Description Exemple
preParseRequest IHttpRequest *, Error & Appelé pour décoder le charset Decodage de la requète
parseRequest Parsing de la requête
postParseRequest IHttpRequest *, Error & Appelé à la fin du parsing du header HTTP URL Rewriting

GetContent

Ce module permet de récupérer et reformater le contenu de la requète

Nom Paramètre(s) Description Exemple
getContent Récuperation du contenu de la requête
contentModule IHttpRequest *, Error & Permet de modifié le contenu de la requète Proxy

GenerateResponse

Ce module permet de générer le contenue de la réponse ainsi que de la formater

Nom Paramètre(s) Description Exemple
CGIModule IHttpResponse *, Error & Gestion des CGI Module CGI
generateResponse Génération du contenu de la réponse
postGenerateResponse IHttpResponse *, Error & Appelé pour encoder le contenu de la réponse Encodage de la réponse

SendResponse

Ce module permet d'envoyer la reponse au client

Nom Paramètre(s) Description Exemple
preSendRequest IHttpResponse *, Error & Gestion du SSL SSL
SendRequest Envoi de la requête
